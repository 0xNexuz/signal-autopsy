How Signal Autopsy works.

Signal Autopsy is a browser-based product prototype for pre-trade failure intelligence. It does not place orders. It turns an AI trading agent's thesis into a visible pre-mortem, guardrail, and audit event.

1. Input Model

The replay lab asks for market, strategy, confidence, leverage, 24h drift, and depth quality. Those inputs generate the live thesis and the failure map. The thesis changes as the controls change, so the agent story always matches the selected conditions.

2. Fragility Score

The score is a synthetic risk indicator for the prototype. It combines directional drift, confidence pressure, requested leverage, and weak depth. Higher scores mean the trade idea is more fragile before execution.

fragility = 24
  + abs(24h_drift) * 4.2
  + max(0, confidence - 58) * 0.45
  + leverage * 1.72
  + max(0, 58 - depth_quality) * 0.48

3. Route Decisions

0-37Sandbox allow.
38-54Guarded micro-test.
55-71Paper or probe only.
72+Block live routing.

4. Guardrail Compiler

The guardrail output converts the verdict into machine-readable execution constraints: max leverage, live route permission, re-check timing, cancellation triggers, and the current thesis. A real implementation would pass that policy into an agent runner, exchange connector, or approval workflow.

5. Ledger

The ledger records intentional pre-mortem runs only. Slider previews update the analysis but do not write audit rows. This keeps the ledger reserved for decisions that the operator actually submitted.

6. Bitget Integration Path

The current build is static. The next production step is to connect market depth, ticker, funding, and order-routing data through the Bitget API. Start from the official docs linked in the product footer.
